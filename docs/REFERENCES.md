<h1 align="center">Referencing Brand Assets</h1>

The **Brand Assets** repository is the canonical source for all official Lupaxa branding. Rather than copying assets into individual repositories, projects
are encouraged to reference them directly from this repository whenever practical.

Using shared assets ensures that branding remains consistent across **The Lupaxa Project** ecosystem while reducing duplication and simplifying future updates.

## Repository URL

The official repository is:

```text
https://github.com/the-lupaxa-project/brand-assets
```

Raw assets are available from:

```text
https://raw.githubusercontent.com/the-lupaxa-project/brand-assets/master/
```

## Raw Asset URLs

General format:

```text
https://raw.githubusercontent.com/the-lupaxa-project/brand-assets/master/logos/organisations/<organisation>/<asset>
```

Example:

```text
https://raw.githubusercontent.com/the-lupaxa-project/brand-assets/master/logos/organisations/aws-toolbox/readme-logo.png
```

## Best Practices

When referencing branding assets:

- Always reference the shared repository where practical.
- Avoid duplicating assets across repositories.
- Use `readme-logo.png` for repository headers.
- Use `logo-transparent.png` for documentation and websites.
- Use `logo-white.png` on dark backgrounds.
- Prefer the highest practical resolution.
- Preserve aspect ratios when resizing.
